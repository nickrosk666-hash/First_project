# Проверенные фреймворки оценки SaaS-идей

## 1. Paul Graham (YC) — "How to Get Startup Ideas"

### Ключевые принципы
1. **Лучшие идеи = проблемы, которые есть у основателя лично**
2. **"Плохие" идеи часто лучшие** — идеи, которые выглядят плохо для большинства, но работают (Airbnb, Dropbox)
3. **Ищи "organic" идеи** — те, что возникают из реальной жизни, а не из brainstorming
4. **Три фильтра**: Основатель хочет это сам? + Может построить? + Мало кто осознаёт, что это нужно?

### Красные флаги YC
- "Решение в поисках проблемы" — технология есть, проблемы нет
- "Мы сделаем как X, но для Y" без глубокого понимания Y
- Идея нравится основателю, но никто за неё не заплатит
- Рынок кажется огромным, но нет конкретного первого клиента

### Зелёные флаги YC
- Основатель = целевой клиент
- Уже есть люди, которые решают проблему вручную или костылями
- Конкуренты существуют, но плохие (= подтверждённый спрос)
- Идея вызывает реакцию "почему этого ещё нет?"

### Как применить к автоматическому скорингу
- **Proxy для "organic idea"**: высокий engagement на Reddit/HN + конкретная боль
- **Proxy для "основатель = клиент"**: мы ищем идеи в наших нишах (developer tools, SaaS)
- **Proxy для "костыльное решение"**: посты типа "I'm using Excel/Notion/Airtable for..."

## 2. Rob Walling (MicroConf) — Stair-Step Approach

### Три ступени
1. **Ступень 1**: Одноразовый продукт (плагин, шаблон, курс) — $500-$2K/мес
2. **Ступень 2**: SaaS с подписками — $5K-$50K MRR
3. **Ступень 3**: Масштабируемый SaaS-бизнес — $50K+ MRR

### Критерии для оценки (Rob Walling)
| Критерий | Вопрос |
|----------|--------|
| Audience | Есть ли уже собранная аудитория, которой можно продавать? |
| Pain level | Насколько острая боль? (aspirin vs vitamin) |
| Willingness to pay | Люди уже платят за альтернативы? |
| Market size | Достаточно ли клиентов при цене $X? |
| Competition | Конкуренты — это хорошо (спрос есть), плохо если 10+ крупных |
| Founder-market fit | Понимаешь ли ты рынок и клиента? |

### Как применить
- **Добавить в скоринг**: `pain_level` — aspirin (10) vs vitamin (3)
- **Добавить**: `willingness_to_pay` — люди уже платят за альтернативы?
- Наша модель пока не учитывает "острота боли" — это критично

## 3. "Hair on Fire" фреймворк

### Концепция
Представь: у человека горят волосы. Он купит ЛЮБОЙ огнетушитель, неважно какого цвета.

**Уровни боли:**
- 🔥🔥🔥 **Hair on Fire** (10): Критическая проблема, нужно решение СЕЙЧАС (compliance deadline, security breach)
- 🔥🔥 **Hard Problem** (7): Ощутимая боль, но живут с ней (неэффективные процессы)
- 🔥 **Nice to Have** (4): Было бы неплохо, но не критично (dashboard-ы, аналитика)
- ❄️ **Vitamin** (2): Никто не просит, основатель думает что нужно

### Как определить уровень из данных
- **Hair on Fire**: Reddit посты с "urgent", "need immediately", "deadline", "emergency"
- **Hard Problem**: "frustrated with", "wasting hours on", "manual process"
- **Nice to Have**: "would be cool if", "it would be nice"
- **Vitamin**: Нет запросов от рынка, только предположения

## 4. Temporal Analysis — тайминг входа на рынок

### Кривая тренда
```
        ╱╲
       ╱  ╲         ← Peak (ОПАСНО: хайп)
      ╱    ╲
     ╱      ╲───── ← Plateau (ХОРОШО: стабильный спрос)
    ╱
   ╱ ← Early Rise (ЛУЧШЕ ВСЕГО: растущий спрос, мало конкурентов)
  ╱
──╱ ← Too Early (ПЛОХО: рынок не готов)
```

### Сигналы по стадиям (Google Trends + Reddit)

| Стадия | Google Trends | Reddit | Действие |
|--------|--------------|--------|----------|
| Too Early | <10 interest, спорадические | 0-2 поста, без engagement | SKIP |
| Early Rise | 10→30, рост 3 мес+ | 5-20 постов/мес, растут | **BUILD** |
| Growth | 30→60, устойчивый рост | 50+ постов/мес, активные | BUILD (спешить) |
| Peak/Hype | 80-100, резкий скачок | Сотни постов, мемы | ⚠️ Осторожно |
| Plateau | 40-60, стабильно | Устойчивое сообщество | BUILD (ниша) |
| Decline | Падение >20% за квартал | "is X dead?", уход лидеров | SKIP |

## 5. Обновлённая модель скоринга для Autonomy

### Старая модель (6 критериев)
Хороша, но пропускает критические сигналы.

### Новая модель (8 критериев)

| # | Критерий | Вес | Что оцениваем | Источник данных |
|---|----------|-----|---------------|-----------------|
| 1 | market_size | 0.15 | Размер рынка, рост | Google Trends, TechCrunch |
| 2 | automation_feasibility | 0.20 | Можно ли без людей | Claude анализ |
| 3 | pain_level | 0.15 | 🔥 Hair on Fire vs Vitamin | Reddit sentiment |
| 4 | competition | 0.10 | Свобода ниши | GitHub, ProductHunt |
| 5 | willingness_to_pay | 0.10 | Уже платят за альтернативы? | ProductHunt pricing, SaaS reviews |
| 6 | margin_potential | 0.10 | Маржинальность | Claude анализ |
| 7 | build_complexity | 0.10 | Скорость MVP | Claude анализ |
| 8 | timing | 0.10 | Стадия тренда (Early Rise = 10) | Google Trends кривая |

### Изменения от старой модели
- **Добавлено**: `pain_level` (0.15) — самый важный предиктор по YC и MicroConf
- **Добавлено**: `willingness_to_pay` (0.10) — "люди уже платят" = валидация
- **Добавлено**: `timing` (0.10) — стадия тренда критична для входа
- **Уменьшено**: `automation_feasibility` 0.25→0.20 (всё ещё важен, но не единственный)
- **Убрано**: `recurring_revenue` (0.10) — поглощён `willingness_to_pay` + `margin`
- **Итого**: 8 критериев, сумма весов = 1.0

### Обновлённая формула
```
composite = Σ(score_i × weight_i) для i = 1..8
```

### Обновлённые вердикты
| Composite | Вердикт | Действие |
|-----------|---------|----------|
| **8.0+** | 🟢 **BUILD** | → Validation Layer |
| **6.5–7.9** | 🟡 **BET** | Telegram + ручной обзор |
| **4.5–6.4** | 🟠 **FLIP** | Сохранить, пересмотреть через 2 нед |
| **<4.5** | 🔴 **KILL** | Отбросить |

## 6. Валидация за $0

### Метод 1: Reddit Signal Strength
- Посчитать (upvotes × 2) + (comments × 3) для постов о проблеме
- >100 = сильный сигнал, >500 = "Hair on Fire"

### Метод 2: Google Trends Slope
- Рост >30% за 3 месяца + объём >20 = Early Rise
- Рост >100% за 1 месяц = возможно хайп (проверить через Reddit)

### Метод 3: "Duct Tape" тест
- Люди решают проблему с помощью Excel, Notion, Zapier + 5 сервисов?
- Посты "my workflow for X" с 10+ шагов = сильный сигнал для SaaS

### Метод 4: Конкурент-аудит
- 1-3 конкурента с плохими отзывами = ОТЛИЧНО (спрос есть, решение плохое)
- 0 конкурентов = возможно рынка нет
- 10+ конкурентов = нужна сильная дифференциация
